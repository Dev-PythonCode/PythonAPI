The Learning Paths API provides career roadmap system with curated learning
resources for technologies required in job openings. It enables integration with
external learning portals (Udemy, Coursera, LinkedIn Learning, etc.) for
dynamic course recommendations.

Architecture:
  - Data Source: data/learning_paths.json (extensible structure)
  - API Server: Python Flask (port 5000)
  - Frontend: Blazor/Razor components (.NET)
  - Mode: Stateless REST API

Design Principles:
  âœ“ Extensible data format for future API integrations
  âœ“ Multiple difficulty levels (Beginner, Intermediate, Advanced)
  âœ“ Rich metadata about providers and costs
  âœ“ Support for batch technology lookups
  âœ“ Response caching support

Fields Explanation:
  - display_name: Human-readable technology name
  - category: Technology category (used for filtering)
  - difficulty_levels: Supported skill levels
  - learning_resources: Organized by difficulty level
    - title: Course title
    - provider: Platform providing the course
    - url: Direct link to course
    - duration: How long the course takes
    - level: Skill level (Beginner/Intermediate/Advanced)
    - cost: Price or free indicator
    - rating: Average rating (1-5 stars)
    - description: What the course covers

â“ FAQ & TROUBLESHOOTING
================================================================================

Q: How do I add a new technology?
A: Edit data/learning_paths.json and add a new entry under "technologies" with
   the required structure. The API will recognize it immediately on next request.

Q: Can I use different providers than listed?
A: Yes! The data structure supports any provider. Add entries with your desired
   provider in the learning_resources section.

Q: How do I integrate with Udemy/Coursera?
A: Create a new service file (external_learning_api.py), implement API calls to
   the external service, and merge results with static data. See integration
   notes above for implementation pattern.

Q: How do I cache API results?
A: Implement caching middleware in Flask with Redis or in-memory cache. TTL of
   1 hour is recommended for learning resources that don't change frequently.

Q: What if a technology isn't found?
A: The API returns it in "not_found" array. You can then add it to
   learning_paths.json or integrate an external API that provides it.

Q: Can I limit courses by cost?
A: Currently, cost information is in the response. Filter on client side or add
   a "max_cost" parameter to the API request for server-side filtering.

Q: How do I update course data?
A: Edit learning_paths.json directly, or integrate external APIs to pull live
   data. Remember to restart the Flask app if editing the file.

Q: Is there a limit on technology queries?
A: No hard limit currently, but for production add rate limiting. Recommended:
   100 requests/minute per IP.
