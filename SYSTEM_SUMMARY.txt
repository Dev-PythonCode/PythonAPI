ARCHITECTURE TRANSFORMATION
================================================================================

BEFORE (Rule-Based System):
  • Hardcoded role normalizations (testing engineer → software testing engineer)
  • Hardcoded tech-to-career mappings (selenium → software testing engineer)
  • Special handling for specific scenarios (iOS, Android platforms)
  • Multiple conditional statements and regex patterns
  • Brittle and difficult to extend
  • Required code changes for new roles/technologies

AFTER (Pure NLP-Based System):
  • Parser extracts entities via NER model (roles, technologies, skills)
  • 6-tier matching strategy with scoring
  • Token-based similarity for flexible matching
  • Multi-factor scoring combining all strategies
  • All behavior driven by training data
  • Easily extensible through JSON updates only
  • Deterministic and reproducible results

================================================================================
MATCHING STRATEGIES (In Order of Execution)
================================================================================

Strategy 1: Exact Entity Matching
  Input: Parsed roles from NER → "full-stack developer"
  Action: Check if matches any training key exactly
  Result: Immediate return if match found

Strategy 2: Substring Matching
  Input: Parsed entities vs training keys
  Action: Check for partial matches with score calculation
  Result: Score based on match length

Strategy 3: Technology Matching
  Input: Parsed technologies → [React, Node.js]
  Action: Check training entries' technology lists
  Result: High score if technologies match

Strategy 4: Multi-Skill Matching
  Input: 2+ parsed skills
  Action: Find training entries with most skill matches
  Result: Return if ≥2 skill matches found

Strategy 5: Token-Based Similarity (Jaccard Index)
  Input: Prompt vs training keys
  Action: Calculate word overlap percentage
  Result: Order-insensitive matching
  Example: "SQL Database Engineer" matches "SQL Engineer"

Strategy 6: Category Matching
  Input: Parsed categories from tech dictionary
  Action: Match against training entry tech/skills
  Result: Score-based ranking

Fallback: Generic Roadmap
  Input: No matches found
  Action: Return 'generic' career entry
  Result: Basic learning roadmap structure
